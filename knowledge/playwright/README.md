# Playwright Knowledge

This module contains Playwright-specific knowledge used by QA Brain to evaluate test automation quality.

The goal is not to teach Playwright syntax.

Instead, this knowledge helps QA Brain understand whether a Playwright test follows industry best practices.

---

# Scope

This module covers:

- Auto Waiting
- Locators
- Assertions
- Fixtures
- Test Isolation
- Retries
- Parallelism
- Project Structure
- Page Object Model
- Network Testing

---

# How QA Brain Uses This Module

QA Brain loads Playwright knowledge after:

1. Identifying the testing framework.
2. Determining the tested feature.
3. Selecting the required QA techniques.

Playwright knowledge is combined with:

- ISTQB
- OWASP
- Unicode
- Microsoft Testing
- Google Testing

to generate comprehensive reviews.

---

# Design Principles

This module is based on:

- Official Playwright Documentation
- Official Playwright Best Practices
- Community best practices
- QA engineering principles

---

# Goal

The purpose of this module is to enable QA Brain to review Playwright tests like an experienced Senior Test Automation Engineer.
